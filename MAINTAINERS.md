## Git attribution

The first local commits were created before Git author details were configured and appear as `Tu Nombre <tu-correo-de-github@example.com>`. `.mailmap` maps those commits to `@chrisbacale` for contributor attribution without rewriting public history.

## Maintainer responsibilities

- Review pull requests and issues.
- Keep official-source references current.
- Ensure official data, derived metrics, and simulations remain clearly labelled.
- Run `npm run verify` before releases.
- Avoid publishing personal data, credentials, or unverified electoral claims.

## Becoming a maintainer

This project is early-stage. New maintainers should first contribute small documentation, source-validation, test, or accessibility improvements. Maintainer access requires a consistent record of careful civic-data handling and respectful collaboration.
